    3. When did Error 404 first came out?

      The Error 404 "Not Found" message has been around since the early days of the World Wide Web. It originated with the HTTP 0.9 and 1.0 specifications, which were published in 1992 by Tim Berners-Lee and the IETF (Internet Engineering Task Force). The first official documentation of HTTP 1.0, including status codes like 404, appeared in RFC 1945 (May 1996).

      So while the concept of "404 Not Found" existed in 1992, it became officially standardized in 1996.

  2. The Tragic Tale of a 404 Page

    1. The Request Begins

      • A user types a URL into their browser or clicks a link.
      • The browser sends an HTTP request to the web server, asking for a specific resource (a webpage, image, or file).

    2. Server Checks Its Map

      • The web server receives the request and checks its file system or database to see if the requested file exists.
      • If the file is found, the server happily sends it back with an HTTP 200 OK status.
      • If the file is missing, things start to go very wrong...

    3. The Horror Unfolds

      • The server realizes the requested resource does not exist or has been moved without a redirect.
      • Since it has no valid response, it generates an HTTP 404 Not Found status.

    4. The 404 Page Awakens

      • Instead of an empty response, the server loads the custom 404 page (if one exists).
      • The browser receives this sad little error page, which tells the user:
        • ❝Sorry, what you're looking for is gone! Maybe it never existed? Maybe it was deleted? Maybe the link is broken? Who knows!❞

    5. The User's Fate

      • The user either:
        • Checks their URL for typos.
        • Hits the back button in frustration.
        • Leaves the website forever.

    6. The Server Logs the Failure

      • The server might log the missing request, allowing the admin to see which pages are broken.
      • If the website owner is proactive, they might set up a redirect (301/302) so future users won’t hit the same dead end.

  3. Why Does 404 Happen?

    • The page was deleted but not redirected.
    • The user typed the wrong URL.
    • The page was moved without updating links.
    • The link was broken (bad coding, outdated bookmarks, etc.).
    • A network issue messed up the request.
